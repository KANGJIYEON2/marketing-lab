# revenue-impact

매출에 직접 영향을 주는 자동화 3종. **광고 효율 + 리드 전환 + LP 전환** 직격.

## 이 카테고리의 목적

콘텐츠가 아무리 좋아도 전환 단계에서 새면 매출이 안 생긴다. 광고 카피, 리드 응대, 랜딩페이지 — 3개 지점의 전환율을 자동으로 끌어올린다.

## 포함 자동화

| # | 자동화 | 핵심 KPI |
|---|---|---|
| [06](./06-ad-evolution/) | 광고 카피 진화 엔진 | ROAS, 광고 피로도 |
| [07](./07-lead-sequence/) | 리드 캡처 → 시퀀스 | 리드 → MQL 전환률 |
| [10](./10-lp-ab-test/) | LP A/B 자동 실험 | LP CVR |

## 데이터 흐름

```
[content-machine 산출물]
    │
    ├─▶ [06 광고 진화] ─ Meta/Google에 변형 자동 업로드
    │
    ├─▶ [10 LP A/B]   ─ 헤드라인·CTA 변형 자동 배포
    │
    ▼
[유료 트래픽] ─▶ [폼 제출] ─▶ [07 리드 시퀀스] ─▶ 영업·전환
                                    │
                                    ▼
                              CRM (Notion: Leads)
```

## 의존성

- **06**: content-machine 02(콘텐츠 자산)에서 광고 카피 시드 가져옴. 06 단독으로도 동작 가능하나, 02가 있으면 품질↑
- **07**: 독립 실행 가능. 이커머스/SaaS 모두 적용
- **10**: 광고 트래픽이 일정량 이상 있어야 통계적 유의성 확보 (월 1만 세션 권장)

## 구축 순서 (권장)

1. **07 먼저** — 리드 새는 거 막는 게 최우선 (트래픽 규모 무관)
2. **06** — 광고 운영 중이면 즉시 ROI
3. **10** — 트래픽 규모 확보 후
