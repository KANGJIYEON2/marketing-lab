# tone-guide-by-channel

채널별 답변 톤 가이드. `inbox-classify-reply.md`의 시스템 메시지에 임베드되어 있고, 이 문서는 가이드를 어떻게 짤지에 대한 메타 설명.

## 채널별 톤 매트릭스

| 채널 | 길이 | 형식성 | 이모지 | 시그니처 | 예시 |
|---|---|---|---|---|---|
| `instagram_comment` | 1-3 문장 | 낮음 | 최대 1개 | 없음 | "감사해요! 다음 글도 기대해주세요 🙌" |
| `instagram_dm` | 1-3 문장 | 낮음 | 최대 1개 | 없음 | "안녕하세요! 어떤 부분 도와드릴까요?" |
| `gmail` | 2-5 문단 | 높음 | 없음 | 가능 | "안녕하세요, 문의 주셔서 감사합니다. \n\n[본문]\n\n감사합니다.\n[이름]" |
| `youtube_comment` | 2-4 문장 | 중간 | 최대 1개 | 없음 | "좋은 질문이에요! [답변]" |
| `blog_comment` | 2-4 문장 | 중간 | 없음 | 없음 | "댓글 감사합니다. [답변]" |
| `x_mention` | 1-2 문장 | 낮음 | 가능 | 없음 | 짧고 재치 있게 |

## 인텐트별 톤 조정

분류된 `intent`에 따라 위 톤을 변주:

### `question`
- 직접 답이 있으면 → 즉답 + 추가 자료 링크
- 답이 길어지면 → DM/이메일로 이동 제안

### `complaint`
- **무조건 사과 먼저** ("불편을 드려 죄송합니다.")
- 그 다음 사실 확인 ("주문 번호/날짜 알려주시면…")
- 마지막에 해결 방안

### `praise`
- 감사 표현 진심으로 (반복 X)
- 다음 콘텐츠/제품 살짝 예고 가능

### `sales_lead`
- 즉시 응답 (urgency=high)
- 다음 액션 제안 (미팅·자료·통화)
- 영업 톤 X — 도움 톤 O

### `spam`
- `draft_reply` 비움
- `status=ignored`로 직행 (v2에서 자동화)

### `other`
- 일반 친절한 응답
- 분류 어려우면 `⚠️ 사람 확인 필요` 부착

## 언어 매칭

- 입력 메시지 언어 그대로 답변 (한국어 → 한국어, 영어 → 영어)
- 혼용 (예: 한·영 섞임) → 메시지의 주 언어로

## v2: 톤 학습

지금은 정적 가이드. 향후:
1. 사람이 발송 전 `draft_reply` 수정 → diff를 누적 저장
2. 한 달 후 패턴 추출 → 시스템 프롬프트에 `Frequent corrections:` 섹션 자동 주입
3. LLM이 사용자 스타일 점차 학습

이건 PRD v2 항목으로 분리.

## 디버깅 팁

답변이 어색하면:
1. 어느 채널인지 확인 → 채널 톤 가이드 재확인
2. brand_voice가 충분히 구체적인지 ("친근하지만 전문적"보다 "20대 1인 창업자처럼 1인칭, 자조적 농담 가능, 영업톤 X"가 낫다)
3. 답변 예시 2~3개를 시스템 메시지에 few-shot으로 추가
