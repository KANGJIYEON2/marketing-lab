# content-machine

콘텐츠 생산·확산 자동화 4종. **글감 발굴 → 생산 → 최적화 → 자산화** 풀 사이클.

## 이 카테고리의 목적

"오늘 뭐 쓰지?" 질문을 없애고, 한 번 만든 콘텐츠를 N개 채널에서 활용한다. 콘텐츠는 다른 자동화(광고·LP·이메일)의 input이 되는 가장 상위 자산.

## 포함 자동화

| # | 자동화 | 핵심 산출물 |
|---|---|---|
| [01](./01-idea-engine/) | 콘텐츠 아이디어 엔진 | 페인포인트 DB → 글감 큐 |
| [02](./02-youtube-factory/) | 유튜브 → 멀티채널 팩토리 | 영상 1편 → 콘텐츠 5~7개 |
| [05](./05-seo-hunter/) | SEO 11~20위 헌터 | 업데이트 액션 아이템 |
| [08](./08-ugc-collector/) | UGC·리뷰 컬렉터 | 캐러셀·광고·LP 소재 |

## 데이터 흐름

```
[01 아이디어 엔진]
    │ 글감 (Notion: ContentIdeas)
    ▼
[02 유튜브 팩토리] ─ 영상/스크립트 ─▶ 블로그·SNS·뉴스레터 (Notion: ContentDrafts)
    │                                        │
    │                                        ▼
[05 SEO 헌터] ─ 기존 글 업데이트 액션 ◀──── 발행된 콘텐츠
    │
    ▼
[08 UGC 컬렉터] ─ 외부 UGC ─▶ 광고·LP 소재 (Google Drive)
```

## 구축 순서 (권장)

1. **01 먼저** — 다른 자동화가 활용할 글감 DB 확보
2. **02** — 가장 ROI 높은 콘텐츠 증폭기
3. **05** — 기존 글 자산이 쌓인 뒤 (3~4주차)
4. **08** — 발행과 멘션이 어느 정도 쌓인 뒤
